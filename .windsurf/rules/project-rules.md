## Общее

| Что               | Кратко                                                   |
| ----------------- | -------------------------------------------------------- |
| **Бандлер**       | Vite 7 (`npm run dev / build / preview`)                 |
| **Фреймворк**     | React 19 (только функциональные компоненты и хуки)       |
| **UI‑библиотека** | MUI v7 (`@mui/material`, `@mui/icons-material`, Emotion) |
| **Язык**          | комментарии и описания — **на русском**          |

## 1. Архитектура проекта

1. Исходники находятся в `src/`. Каждый **компонент** — в отдельной папке `<ComponentName>` вместе со стилями.

## 2. React

* **Только** функциональные компоненты, без class.
* PascalCase для имён файлов и компонентов (`ProductCard.jsx`).
* Props неизменяемы; стейт всегда обновляем через set‑функции.
* Внешние эффекты — в `useEffect` с корректным списком зависимостей.
* Обёртка `<React.StrictMode>` обязательна (см. `main.jsx`).

## 3. MUI (Material UI 7)

* Импортируй компоненты из `@mui/material`, иконки — из `@mui/icons-material`.
* Стили: `sx` prop или `@emotion/styled`. **Не** использовать `makeStyles`/`styled-components`.

## 4. Ни в коем случае не добавляй:
- стейт-менеджеры
- Tailwind
- не пиши стили инлайново в файле компонента на элементах
- не используй !important, добавляй спицефичность через каскад стилей
- не пиши лишних классов в css-правилах, используй каскад достаточный для применения стилей
- используй только кастомные css-классы, не стилизуй через классы библиотек, если есть весомая причина нарушить это правило, уточняй и жди подтверждения

## 5. Правила для AI‑ассистента

1. **Всегда** уточняй требования, если есть сомнения.
2. Отвечай только изменённым кодом или diff без лишнего текста.
3. Не меняй файл, если изменение не связано с запросом.
4. Соблюдай структуру и правила этого документа.
5. Не добавляй зависимости без запроса.
6. Сохраняй совместимость с React 19 и Vite 7.
7. Комментарии к коду — на русском, кратко и по делу.
8. Если решение объёмное, предлагай его частями и жди подтверждения.

## 6. Зависимости

- Всегда добавляй фиксированные (concrete) версии зависимостей без символа ^ или ~.
- Не добавляй новые зависимости без запроса.
- Если нужно добавить зависимость, уточни и жди подтверждения.

## 7. Тесты

- Все компоненты должны иметь тесты.
- Тесты должны быть написаны на Vitest и React Testing Library.
- Тесты должны проверять все возможные сценарии работы компонента, включая граничные случаи, но только те, которые вилит пользователь.
- Тесты должны быть написаны в отдельном файле с названием `ComponentName.test.jsx`.
- Дочерние компоненты, хуки, провайдеры и другие зависимости должны быть заменены на моки в тестах.
- Всегда запускай тесты перед отправкой изменений, при необходиости исправляй падающие тесты.

## 8. JSDoc

- Все функции и компоненты должны иметь JSDoc комментарии.
- JSDoc комментарии должны быть на русском языке.
- JSDoc комментарии должны содержать описание функции, параметров и возвращаемого значения.
- JSDoc комментарии должны быть написаны в формате TypeScript.
- JSDoc должен содержать только комментарии для основныъ функций, внутренности комментировать не надо.

## 9. README.md

- Всегда обновляй README.md при добавлении новых зависимостей, изменении конфигурации или добавлении новых функций.
- README.md должен содержать описание проекта, инструкцию по установке и запуску, а также список зависимостей.
- Ведите changelog для отслеживания изменений.
