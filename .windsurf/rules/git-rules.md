# Правила работы с Git

Данный документ описывает соглашения по написанию коммитов в проекте TechStore, основанные на спецификации [Conventional Commits 1.0.0](https://www.conventionalcommits.org/en/v1.0.0/).

## 📋 Формат коммита

```
<type>[optional scope][!]: <subject>

[optional body]

[optional footer(s)]
```

## 🏷️ Типы коммитов

| Тип | Описание | Пример |
|-----|----------|--------|
| `feat` | Новая функциональность | `feat(cart): добавить валидацию промокодов` |
| `fix` | Исправление бага | `fix(products): исправить отображение скидок` |
| `docs` | Изменения в документации | `docs(readme): обновить инструкцию по установке` |
| `style` | Правки по стилю кода (без изменения логики) | `style(components): применить единое форматирование` |
| `refactor` | Рефакторинг кода | `refactor(hooks): вынести логику корзины в отдельный хук` |
| `test` | Добавление или исправление тестов | `test(cart): добавить тесты для валидации промокодов` |
| `chore` | Обслуживание проекта (зависимости, конфиги) | `chore(deps): обновить React до версии 19` |

## 🔧 Scope (область)

Указывает часть проекта, которую затрагивает изменение:

- `cart` — корзина покупок
- `products` — каталог товаров
- `ui` — UI компоненты
- `api` — API и работа с данными
- `hooks` — React хуки
- `utils` — утилиты
- `config` — конфигурация проекта
- `styles` — стили (CSS/MUI)

## ✏️ Subject (заголовок)

- Используйте настоящее время, повелительное наклонение
- Не используйте заглавные буквы в начале
- Не ставьте точку в конце
- Максимум 50 символов

✅ Правильно:
```
feat(cart): добавить валидацию промокодов
```

❌ Неправильно:
```
feat(cart): Добавил валидацию промокодов.
```

## 💥 Breaking Changes (критические изменения)

Указываются через `!` в заголовке или `BREAKING CHANGE` в footer:

```
feat(api)!: изменить формат ответа корзины

BREAKING CHANGE: поле `total` теперь возвращается как строка с валютой
```

## 📖 Примеры для проекта TechStore

### Добавление функциональности
```
feat(cart): добавить ограничение количества товара (макс. 2 шт.)
feat(promo): реализовать применение скидки по промокоду
feat(products): добавить фильтрацию по категориям
```

### Исправление багов
```
fix(cart): исправить расчет итоговой суммы при удалении товара
fix(ui): поправить отображение кнопки на мобильных устройствах
fix(products): исправить загрузку изображений товаров
```

### Рефакторинг
```
refactor(cart): вынести логику подсчета в отдельный хук useCartTotal
refactor(api): унифицировать обработку ошибок
refactor(components): разделить ProductCard на подкомпоненты
```

### Тесты
```
test(cart): добавить тесты для валидации промокодов
test(products): покрыть тестами ProductGrid
test(hooks): протестировать useLocalStorage
```

### Документация
```
docs(readme): добавить раздел с примерами использования
docs(api): описать формат данных корзины
```

## 🔄 Рабочий процесс

1. Создавайте коммиты атомарно — один логический блок изменений
2. Пишите понятные сообщения, отвечающие на вопрос "что делает этот коммит?"
3. Используйте `git commit -m "type(scope): subject"` для простых изменений
4. Для сложных изменений используйте `git commit` с редактором для body

## 📝 Полный пример коммита

```
feat(cart): добавить валидацию промокодов

Реализована проверка промокодов на сервере:
- Валидация формата промокода
- Проверка срока действия
- Ограничение количества использований

Closes: #123
```
